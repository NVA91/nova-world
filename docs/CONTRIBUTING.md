# Contributing

Danke, dass du zum Ansible Test Controller beitragen möchtest!

## Wie du beitragen kannst

### Bugs melden

Wenn du einen Bug gefunden hast, öffne bitte ein Issue mit:

- Eine klare Beschreibung des Problems
- Schritte zum Reproduzieren
- Erwartetes Verhalten
- Tatsächliches Verhalten
- Deine Umgebung (OS, Docker-Version, etc.)

### Feature-Anfragen

Wenn du ein neues Feature vorschlagen möchtest, öffne bitte ein Issue mit:

- Eine klare Beschreibung des Features
- Warum es nützlich wäre
- Mögliche Implementierungsansätze

### Code-Beiträge

1. Forke das Repository
2. Erstelle einen Feature-Branch (`git checkout -b feature/my-feature`)
3. Committe deine Änderungen (`git commit -am 'Add my feature'`)
4. Pushe den Branch (`git push origin feature/my-feature`)
5. Öffne einen Pull Request

## Coding-Standards

- Verwende aussagekräftige Commit-Messages
- Schreibe Tests für neue Features
- Halte die Dokumentation aktuell
- Folge den bestehenden Code-Stilen

## Lizenz

Durch das Einreichen von Beiträgen stimmst du zu, dass deine Beiträge unter der gleichen Lizenz wie das Projekt lizenziert werden.
